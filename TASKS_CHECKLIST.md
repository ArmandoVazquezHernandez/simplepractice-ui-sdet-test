Tasks Feature - Happy Path Checklist
  Overview:
   This Checklist cover the main happy path scenarios for the Tak feature in SimplePractice.
   The focus is on validating expected positive behaviors when users interact with the feature.

1.- Navigation:
*Uer can open the Create Task from the main navigation menu
*Tasks page loads successfully without errors
*Existing tasks are displayed correctly

2.-Create Task
*User can Open the "Create Task" form
*User can create a task using valid input data
*System successfully save the new task
*Newly created task appears in the task list
*Task title is displayed correctly
*Task persists after page refresh

3.-Task Viibility:
*Created task is visible in the list of tasks
*Task information (Title, tatus) is displayed correctly
*Tasks are organized and readable in the UI

4.- Complete Task:
*User can mark a tak as "Completed"
*Task status change to completed successfully
*Completed task i visually distinguihable
*Completed task remains completed after page refresh

5.- Task Persitence:
*Task remain saved after navigating away from the page
*Tasks remain visible after refreshing the browser
*Completed tasks mantain their status

SUMMARY:
The Tasks feature should allow users to:
*Create tasks successfully
*View tasks clearly in the UI
*Complete tasks without issues
*Mantain task state across sessions

This checklist focuses exxclusively on "Happy Path Scenarios" as required for the assessment.
